Apache Maven is a software project management and comprehension tool based on the concept of a project object model (POM). Maven can manage a project's build, reporting and documentation from a central piece of information.

As we all know, Maven is great tool which has been working great from many years and made foundation for many big CI/CD tools like Jenkins etc.

Still Maven is taken as a command line tool which just produce standard text log. So it's time for Maven to use it as a complete platform rather than a tool. Also the plugins can be more specifically used as apps or extension onto the platform.

The latest solution ( which is still in progress ) propose that maven should be running as a remote service rather that local command line. This would help in consuming it through various channel like a modern UI or a cloud Shell and can be better used as a component for developer workspace servers and cloud IDE like Eclipse Che.

As hosted on M2Hub , there are 3 main components of this stack.

1. M2Hub Server

2. M2Hub UI

3. M2Hub Console

These basic tools can be used to build great services like

- Project Generator,

- Catalog Explorer

- and Source Code Finder etc.

Also after combining it with tools like GitHub it can be used as a

- Code Review and Quality

- Documentation Generator

There are plenties of opportunities and feasibilities once we will start using Maven as a service rather than tool.
